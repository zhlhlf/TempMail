API 文档代码块切换按钮预览

这个单文件原型只演示“每个 API 卡片右上角增加 curl / python 切换”的交互样式，不接入现有业务。每张卡片都展示同一组接口内容，但切换控件风格不同，方便你先挑视觉方向。

单文件 HTML 4 种按钮版本 可直接本地预览
Version A · Segmented Pills

胶囊分段切换

最稳妥，和当前系统按钮风格最接近，识别成本低，适合直接落到现有 API 文档页。推荐优先

📌 获取留存邮件列表（Admin）

GET /api/admin/retained-mails — 仅管理员可调用，分页查看系统留存邮件列表。


            

          

          
特点：视觉稳定、命中区域大、移动端也好用。缺点是会稍微占一点横向空间。

        

      


      

        

          
Version B · Underline Tabs

          
轻量下划线标签

          
更像文档站，视觉更轻，适合想让代码块本身成为主角、按钮存在感更低的场景。

        

        

          

            

              

                

                  
⟳ 续期已过期邮箱

                  

                    
                    
                  

                

                
PUT /api/mailboxes/:id/renew — 仅对已过期邮箱生效，minutes 单位为分钟。

              

            

            

              
              

            

          

          
特点：最像 API 文档产品，界面清爽。缺点是激活态没有胶囊那么显眼。

        

      


      

        

          
Version C · Icon Chips

          
图标标签切换

          
信息密度更高，切换按钮更像“语言卡片”；如果你想让 python 示例更有“高级选项”的感觉，这版会比较讨喜。

        

        

          

            

              

                

                  
📬 创建临时邮箱

                  

                
                
                  

                

                
POST /api/mailboxes — 支持随机地址、随机域名或手动指定 address / domain。

              

            

            

              
              

            

          

          
特点：辨识度强，适合后续扩展到 curl / python / javascript 三种语言。缺点是视觉稍活泼。

        

      


      

        

          
Version D · Hard Split

          
硬切换双按钮

          
更像运维面板或管理后台，按钮对比更强，适合希望“当前代码语言”一眼可见的场景。
